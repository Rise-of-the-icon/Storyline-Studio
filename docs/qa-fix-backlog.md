# QA Fix Backlog

Tracks defects, gaps, and follow-ups surfaced during implementation. Sorted by status, then severity.

## Conventions

- **ID** — stable `QA-###` (never reused). Reference from `implementation-log.md` entries.
- **Status** — `open` · `in-progress` · `blocked` · `done` · `wontfix`. When marking `done`, link the implementation-log entry that closed it.
- **Severity** — `P0` (breaks a demo or violates a gate from `docs/08-AI-SAFETY.md`) · `P1` (visible defect or accessibility issue) · `P2` (polish / DX / nice-to-have).
- **Area** — `tooling` · `design-system` · `screen:S?` · `studio:SS?` · `lib:<module>` · `a11y` · `perf` · `docs`.
- One item per row. Keep the row terse; expand in `implementation-log.md` when the fix lands.
